# TP-Link T1600G-52TS (`192.168.100.15`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Onboard TP-Link T1600G-52TS at `192.168.100.15` as a disabled, read-only SNMP source and prove that it is reachable through the TP-Link collector profile.

**Architecture:** The production source YAML is created by `netctl` with `enabled: false`.  Its community is held only in `/etc/netctl/secrets.env` under an environment variable derived from `secret_ref`; the source test performs SNMP GET/WALK and writes no switch configuration.  The collector timer remains disabled throughout.

**Tech Stack:** Existing `netctl` CLI, PySNMP 7.1.27, systemd, `/etc/netctl/sources.d`, `/etc/netctl/secrets.env`.

## Global Constraints

- SNMPv2c only; never issue SNMP SET.
- Do not echo, commit, inspect, or retain the community value.
- Do not enable `netctl-collect.timer` or any SNMP source for this phase.
- Do not alter OpenVPN, WireGuard, routing, firewall rules, VLANs, ports, or switch configuration.
- Source metadata must be `root:netctl 0640`; only root creates source YAMLs.
- `netctl` must not gain write access to `/etc/netctl/sources.d`.

---

### Task 1: Stage the disabled TP-Link source and protected community

**Files:**
- Create: `/etc/netctl/sources.d/tplink-ito-15.yaml` on production through `netctl`
- Modify: `/etc/netctl/secrets.env` on production through `sudoedit`
- Backup: `/var/lib/netctl/backups/tplink-ito-15-<UTC timestamp>.yaml`

**Interfaces:**
- Consumes: the operator-provided community for `ITO-T1600G-52TS`, without displaying it.
- Produces: disabled source `tplink-ito-15` with `secret_ref` `switch_tplink_ito_15_snmp`.

- [ ] **Step 1: Verify collection is stopped before source creation**

Run:

```bash
systemctl is-active netctl-collect.timer
systemctl is-enabled netctl-collect.timer
```

Expected: `inactive` and `disabled`. Stop if either value differs.

- [ ] **Step 2: Create the source as disabled**

Run:

```bash
sudo /usr/local/sbin/netctl --json sources add-snmp-switch \
  tplink-ito-15 \
  --host 192.168.100.15 \
  --secret-ref switch_tplink_ito_15_snmp \
  --profile-hint tplink \
  --snmp-version 2c \
  --timeout-seconds 2 \
  --retries 1 \
  --max-repetitions 25
sudo chown root:netctl /etc/netctl/sources.d/tplink-ito-15.yaml
sudo chmod 0640 /etc/netctl/sources.d/tplink-ito-15.yaml
sudo install -d -m 0750 /var/lib/netctl/backups
sudo install -m 0640 -o root -g netctl \
  /etc/netctl/sources.d/tplink-ito-15.yaml \
  /var/lib/netctl/backups/tplink-ito-15-$(date -u +%Y%m%dT%H%M%SZ).yaml
```

Expected: the source file exists, its YAML has `enabled: false`, and no collection has run.

- [ ] **Step 3: Enter the community without exposing it**

Run:

```bash
sudoedit /etc/netctl/secrets.env
sudo chown root:netctl /etc/netctl/secrets.env
sudo chmod 0640 /etc/netctl/secrets.env
sudo stat -c '%U:%G %a' /etc/netctl/secrets.env
sudo -u netctl test -r /etc/netctl/secrets.env
```

Add exactly one line in the editor, using the community from the approved inventory:

```text
NETCTL_SECRET_SWITCH_TPLINK_ITO_15_SNMP_COMMUNITY='<community entered only by the operator in sudoedit>'
```

Expected: only `root:netctl 640` metadata is printed; the community is never shown in shell output.

- [ ] **Step 4: Commit the operational evidence without secrets**

Do not commit the production YAML, secret file, backup, IP inventory, or any credential to Git. Record only the source name, profile hint, file metadata, and disabled state in the change record.

### Task 2: Prove read-only TP-Link reachability and leave the source disabled

**Files:**
- Read: `/etc/netctl/sources.d/tplink-ito-15.yaml`
- Read: `/etc/netctl/secrets.env` through the `netctl` process only
- Write: netctl's existing SQLite capability/run records; no FDB collection is executed

**Interfaces:**
- Consumes: `tplink-ito-15` from Task 1.
- Produces: sanitized `sources test` result with TP-Link profile and required capability outcomes.

- [ ] **Step 1: Run the read-only source test**

Run:

```bash
sudo -u netctl /usr/local/sbin/netctl --json sources test tplink-ito-15
```

Expected: `status: ok`, profile `tplink`, and successful required system/interface/bridge capability groups. `success_empty` is acceptable only for optional LLDP/VLAN capabilities.

- [ ] **Step 2: Verify the source remains inert**

Run:

```bash
sudo grep -E '^enabled:' /etc/netctl/sources.d/tplink-ito-15.yaml
systemctl is-active netctl-collect.timer
systemctl is-enabled netctl-collect.timer
systemctl is-active openvpn-web.service
systemctl is-active openvpn-server@server.service
systemctl is-active wg-quick@wg0.service
```

Expected: `enabled: false`; timer `inactive` and `disabled`; all three VPN services `active`.

- [ ] **Step 3: Handle a failed source test without weakening safety**

If the source test reports timeout, authentication/view failure, generic profile, parser error, or a required-group failure, keep the source disabled. Do not retry with SNMP SET, do not enable the timer, and do not change the switch. Retain only the sanitized error category and restore `/etc/netctl/sources.d/tplink-ito-15.yaml` from its backup if its metadata or content was changed.
